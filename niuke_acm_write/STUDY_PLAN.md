# 自变量 Agent 系统工程师 · 牛客网笔试冲刺计划

> 岗位：agent 系统工程师（非算法研究员）  
> 形式：牛客网 ACM 模式，60 分钟，10 道编程题  
> 核心差异：系统工程师考"工程实现 + 具身场景 + 系统集成"，不考模型推导/训练理论

---

## 一、岗位特性分析（决定备考方向）

### agent 系统工程师 vs 大模型算法工程师

| 维度 | 算法工程师 | **系统工程师（你的岗位）** |
|:---|:---|:---|
| **核心能力** | 模型设计、训练、调参 | **系统集成、部署优化、Agent 框架搭建、数据管线** |
| **代码重点** | Attention 变体、Loss 设计 | **工具编排、数据流、调度、延迟优化、多模态融合** |
| **自变量业务** | WALL-A 模型架构设计 | **WALL-A 端侧/云端部署、真机数据闭环、具身思维链工程化** |
| **笔试倾向** | 手撕 Attention、推导梯度 | **函数式工程题：Tokenization、调度、数据清洗、延迟预算、工具链** |

### 自变量技术路线对笔试的映射

| 自变量核心技术 | 笔试映射 | 对应题目 |
|:---|:---|:---|
| 端到端 VLA（WALL-A） | 动作 Tokenization + 多模态输入融合 | **N02, N21, N22** |
| 具身思维链（Embodied CoT） | ReAct 状态机 + 工具调用链编排 | **N03, N24** |
| 真机数据闭环 | 遥操作数据清洗 + 质量过滤 | **N23** |
| 端侧/云端部署 | KV Cache 内存计算 + 延迟预算 + 量化 | **N01, N08, N25** |
| 推理服务 | 请求调度 + 连续批处理 + Block 分配 | **N09, N12, N17** |

**关键结论**：系统工程师笔试 = **40% 具身场景工程题**（动作空间、多模态、数据清洗、工具链）+ **40% Infra 基础题**（内存、调度、量化、延迟）+ **20% LLM 基础题**（Attention、RoPE、Softmax）。

## 四、非编程题可能性分析

| 题型 | 出现概率 | 题数 | 备考方式 |
|:---|:---:|:---:|:---|
| 纯编程 10 题 | **50%** | 10 | 本清单全覆盖 |
| 编程 8 题 + 选择 2 题 | **35%** | 8+2 | 背下方 6 个结论 |
| 编程 7 题 + 选择 3 题 | **15%** | 7+3 | 同上 |

**若出选择题，最可能考点（系统工程师方向）**：
1. **具身动作空间**：机械臂工作空间通常为球体或圆柱体，**z < 0 不可达**
2. **工具调用链**："pick_place" 标准链为 **perceive → grasp → move → release**
3. **延迟预算**：WALL-A 端侧部署要求总延迟 **< 100ms**，瓶颈通常在 **LLM 推理**
4. **数据清洗**：遥操作数据过滤条件：**success=True + 时长正常 + 速度不超安全阈值**
5. **多模态融合**：VLA 模型输入通常按 **视觉 → 语言 → 力觉** 顺序拼接
6. **GQA 选型**：端侧推理优先 **GQA**（减少 KV Cache），而非 MHA

## 七、关键纪律

1. **系统工程师 ≠ 算法工程师**：不要死磕 Attention 推导（N05），优先保具身场景题（N21-N25）。
2. **ACM 输入输出是生死线**：`sys.stdin.readline().strip()` 必须 30 秒内写出。
3. **具身场景题是送分题**：N21-N25 代码量小、逻辑直观，但**必须理解业务场景**（工作空间、工具链、数据清洗）。
4. **边界比核心逻辑更重要**：空输入、除零、越界、恰好贴边（>= vs >）是扣分重灾区。
5. **考前 1 小时只背 8 条口诀**：不再写新代码。
